## 一、核心变化说明

此前设计（deepseek离散分段版）存在边界跳变问题：呼吸系数在比值0.7/1.0/1.4/2.0这几个分段点附近会突然跳变，且每个品种需要9个参数（5档系数+4个边界点），参数越多越容易在回测中过拟合。

本版改为连续线性插值，每个品种只需2个参数（minMult/maxMult），边界点两品种共用，全程无跳变。

以下内容保留自此前定稿、未改变：initial_atr锁定约束、平滑机制（3次采样平均）、XAU整体比ETH更紧的设计意图。以下内容本次改变：呼吸系数计算方式（离散改连续）、冷启动处理方式（手动指定档位改自动代入公式）、XAU的0.8额外收紧层（删除，收紧程度直接体现在min/max数值本身）。

**2026-07-23 参数修正：** 方案原文 XAU `minMult/maxMult = 0.8/1.8` 在历史回测中显著劣于旧离散×0.8；经敏感性扫描后**正式改为 `0.5/1.2`**（见第十节完整数据）。ETH 仍为 `1.2/2.5`。

## 二、核心公式

```
ratioFloor = 0.6      （两品种共用，不需要分别设置）
ratioCeiling = 2.2    （两品种共用，不需要分别设置）

trailDistanceMultiplier(ratio):
    若 ratio <= ratioFloor:   返回 minMult
    若 ratio >= ratioCeiling: 返回 maxMult
    否则:                     返回 minMult + (maxMult - minMult) × (ratio - ratioFloor) / (ratioCeiling - ratioFloor)

其中 ratio = 平滑后的实时ATR / initial_atr
追踪距离 = initial_atr × trailDistanceMultiplier(ratio)
```

### 品种参数（生产锁定）

| 品种 | minMult | maxMult | 追踪距离范围 | 冷启动（ratio=1.0） |
|---|---|---|---|---|
| ETH | 1.2 | 2.5 | 1.2×ATR ~ 2.5×ATR | **1.525** |
| XAU | **0.5** | **1.2** | 0.5×ATR ~ 1.2×ATR | **0.675** |

不再需要XAU的×0.8额外收紧系数——XAU比ETH更紧这个设计意图，已经完全体现在min/max数值本身更小上，不需要额外的乘法层。

> 作废：XAU 0.8~1.8 / 冷启动 1.05（仅作方案草稿；回测证明过松，不得再作生产配置）。

## 三、平滑机制（保留，改为平滑输入而非平滑输出）

关键设计差异：此前版本是"先查表算出各次的系数，再对系数取平均"；本版改为"先平滑ratio这个原始数字，再代入连续公式算一次输出"。后者数学上更干净，不会因为在离散阶梯上取平均而产生奇怪的中间态。

```
实时ATR每5分钟从交易所K线独立计算并采样一次
smoothedRatio = 最近3次采样的实时ATR/initial_atr比值的算术平均
若采样数不足3次，用已有采样数平均（1次用1次，2次取2次平均）
trailDistanceMultiplier(smoothedRatio) 得到最终乘数
```

---

## 四、冷启动处理（0次采样时，自动代入公式，不再需要手动指定）

开仓后第一次实时ATR采样完成之前（0次采样），smoothedRatio直接取1.0（假设市场波动与开仓时持平，这是保守的默认假设）：

```
trailDistanceMultiplier(1.0):
    = minMult + (maxMult - minMult) × (1.0 - 0.6) / (2.2 - 0.6)
    = minMult + (maxMult - minMult) × 0.25

ETH: 1.2 + (2.5-1.2)×0.25 = 1.525
XAU: 0.5 + (1.2-0.5)×0.25 = 0.675
```

空闲/平仓重置时，状态里的 `breathing_coefficient` 必须写成上述冷启动值，**禁止**字面量 `1.0`（否则 ETH 会被夹到 1.2、XAU 会偏松）。
